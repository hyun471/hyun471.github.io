# 클래스와 속성

# 📚 클래스와 속성

## 클래스

- 클래스(class)는 어떤 특성과 기능을 가진 객체를 사용할 수 있게 만드는 설계도와 같다. 
- 클래스는 class + [클래스명] + { 생성자() + 맴버변수 + 메서드() {} }의 형태로 작성된다.
  - 객체 : 클래스를 통해 실제로 작동하는 것
  - 생성자 : 클래스와 같은 이름을 사용하며 객체를 만들 때 자동으로 실행됨
  - 맴버변수 : 변수와 쓰임이 같으며 클래스 안에서 선언되면 맴버변수라고 칭함 (속성)
  - 메서드 : 함수와 기능이 같으며 클래스 안에서 메서드라고 칭함
- 클래스 예시
```dart
//클래스 = Dog
class Dog {
  // 멤버변수 (특성)
  String name = '초코';
  int age = 2;

  // 생성자
  Dog(this.name, this.age);

  // 메서드 (기능)
  void bark() {
    print('$name: 멍멍!');
  }
}

void main() {
  var dog = Dog('코코', 3);
  dog.bark(); 
}

//출력 코코: 멍멍!
```

### 객체

- 클래스라는 설계도를 기반으로 실제 메모리에 만들어진 실체이다.
**즉, 데이터를 변수 또는 함수에 넣고 클래스 기반으로 실행된 것이 객체이다.**

- 변수에 담은 객체 예시 
```dart
class Person {
  String name;

  Person(this.name);

  void greet() {
    print("안녕, 나는 $name이야.");
  }
}

void main() {
  var p = Person("초코"); // ← 클래스 기반 객체
  p.greet();              // → 안녕, 나는 초코이야.
}
```

- 함수에 담은 객체 예시 
```dart
void main() {
  Function greet = () {
    print("안녕, 나는 함수 객체야.");
  };

  greet(); // → 안녕, 나는 함수 객체야.
}
```

### 속성

- 클래스 안에서 작업을 수행할 때 사용하는 데이터이다.
- 즉, 그 객체를 구성하는데 필요한 데이터가 속성이다.
  - 보통 (속성 = 인스턴스 변수 = 멤버 변수) 라고 봐도 된다.
**속성의 종류**

**인스턴스 변수**

- 객체마다 따로 존재하는 속성, 객체를 통해 호출 (Player().name 또는 this.name)
- 클래스 안, 메서드 밖에서 선언한 변수
- 인스턴스 변수 =  멤버 변수
**정적 변수**

- 클래스 전체에서 공유
- 클래스 이름을 통해서만 호출
- static를 붙인 변수
**지역 변수**

- 해당 코드 블록 안에서만 존재하고 사라짐
- 생성자나 메서드 내부에서 선언된 변수
**예시 코드**

```dart
class Player {
  // 인스턴스 변수 (멤버 변수)
  String name = "초코";

  // 정적 변수
  static int playerCount = 0;

  // 생성자
  Player() {
    // 지역 변수
    int bonusHp = 10;
    print("보너스 HP: $bonusHp");
    playerCount++;
  }

  // 메서드
  void greet() {
    // 지역 변수
    String greeting = "안녕, $name!";
    print(greeting);
  }
}
```

### 메서드

- 클래스에 의존하는 함수이다.
**메서드의 종류**

**인스턴스 메서드**

- 객체에 속해 있는 메서드, 객체를 통해 호출 (Player().walk() 또는 this.walk())
- 클래스의 모든 곳에서 접근이 가능
**정적 메서드 (=클래스 메서드)**

- 객체에 종속되지 않고 클래스 자체에 속하는 메서드, 클래스 이름을 통해서만 호출
- 객체를 생성하지 않고 메서드를 호출할 수 있음
- 코드 블록에 인스턴스 변수를 사용할 수 없고 정적 변수만 사용 가능
- static를 붙인 메서드
**예시 코드**

```dart
class Calculator {
 
  int add(int a, int b) {                // 인스턴스 메서드
    return a + b;
  }                                        

  static int multiply(int a, int b) {    // 정적 메서드
    return a * b;
  }
}                                         

void main() {
  Calculator calc = Calculator();        // 인스턴스 메서드는 객체 생성 후 사용
  print(calc.add(3, 4));                 // 출력: 7

  print(Calculator.multiply(3, 4));      // 정적 메서드는 클래스 이름으로 바로 사용
}                                        // 출력: 12

```

### 생성자

- 객체를 생성하고 초기화하기 위해 사용하는 메서드 
- 생성한 객체들은 서로 독립된 객체
**생성자의 종류**

**매개변수 생성자**

- 매개변수를 갖는 생성자
- 매개변수를 통해 인스턴스 변수들의 초기값을 설정
- 매개변수를 갖지 않으면 기본 생성자로 생성된다.
**네임드 생성자**

- 객체를 만들 때, 어떤 특정한 상태(속성값)로 초기화 하고 싶을 때 사용
- 해당 클래스에 특정 이름을 붙여서 객체를 특정한 상태로 만들기 쉬움
- [클래스 이름].[네임드 생성자 이름]([타입] [매개변수 이름]) : this.변수;
**예시 코드**

```dart
class Player {
  String name;

  Player(this.name);        // 매개변수 생성자

  Player.choco() {          // 네임드 생성자
    name = "초코";
  }
}

void main() {
	var p1 = Player("뽀삐");
  var p2 = Player.choco();
  print(p1.name);           // 출력: 뽀삐
  print(p2.name);           // 출력: 초코
}
```

## 상속

- 기존 부모 클래스의 속성과 기능을 새로운 자식 클래스가 물려받는 것
- 부모 클래스에서는 자식 클래스에 있는 속성, 메서드를 사용할 수 없음
- 자식 클래스는 super을 통해 부모 클래스의 속성과 메서드를 사용할 수 있음
- 또한, 자식 클래스는 상속 받은 속성과 메서드를 @Overriding를 통하여 재정의 할 수 있음
  - 변수 재정의 : @override [변수 이름] = [값]; 
  - 함수 재정의 : @override [반환 타입] [함수 이름]() { … }
- 만약 클래스를 상속하고 싶지 않을 때 앞에 final를 붙여 고정 시킴
- class + 자식클래스 + extends + 부모클래스 {  }
## 객체 지향 프로그래밍 (Object-Oriented Programming)

- 객체들을 사용하여 프로그램을 구성하는 방식
- 클래스를 통해 여러 객체를 만들거나, 상속하여 기능 확장을 할 수 있어 코드 재사용을 높일 수 있음
- 또한, 클래스를 통하면 각 객체는 독립되기 때문에 객체를 수정할 때 오류 발생률이 낮음
