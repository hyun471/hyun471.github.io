# 📚 기초 앱 구성 1

## 구성 전 구조 살펴보기

- 플러터로 앱을 만들 때 화면을 구현하기 위해 단계별로 나눠서 구상해야한다.
  1. 레이아웃(화면의 구조 분할하기)
  1. 나누어진 레이아웃 별 필요한 위젯 구상하기
  1. 구현
- 디렉토리 및 파일 분리 (2가지 방식)
  1. 역할 기반 디렉토리 구조
    - 하는 역할에 따라서 파일과 폴더를 분리하는 방식
    - 한눈에 구조 파악이 쉬워 앱이 작을 땐 깔끔하지만, 앱이 커지면 관리하기 어려워짐 
```dart
📁 lib/
├── main.dart
├── 📁 screens/         ← 모든 화면 파일(Home, Store 등)
│   ├── home_screen.dart
│   └── store_screen.dart
├── 📁 widgets/         ← 공통 위젯들
│   └── custom_button.dart
├── 📁 models/          ← 데이터 클래스
│   └── product.dart
├── 📁 providers/       ← 상태관리 클래스
│   └── product_provider.dart
└── 📁 utils/           ← 헬퍼 함수/유틸
    └── date_formatter.dart

```

  1. 기능 기반 디렉토리 구조
    - 한 화면에 보여지는 기능에 따라서 구성 요소별로 파일과 폴더를 분리하는 방식
    - 팀 프로젝트, 실무 앱에서 더 모듈화되고  유지보수가 쉬운 구조, 처음엔 폴더가 많아 복잡해 보임
    - 요즘은 기능 기반 디렉토리 구조가 더 선호 됨. 최신 트렌드
```dart
📁 lib/
├── main.dart
├── 📁 home/
│   ├── home_screen.dart
│   ├── feed.dart
│   ├── home_model.dart
│   └── home_provider.dart
├── 📁 store/
│   ├── store_screen.dart
│   ├── product_card.dart
│   ├── store_model.dart
│   └── store_provider.dart
├── 📁 common/
│   ├── 📁 widgets/
│   │   └── custom_button.dart
│   └── 📁 utils/
│       └── date_formatter.dart

```

### debug 배너 해제

- 개발 환경(debug 환경) 시 기본적으로 나온다.
- MaterialApp의 debugShowCheckedModeBanner속성을 false로 주면 지워줄 수 있다.
```dart
MaterialApp(
      debugShowCheckedModeBanner: false,)
```

## Store app 화면 구성하기

### 컴포넌트화

- 레이아웃
  - 전체적으로 세로로 배치(Column 사용)
- 나누어진 레이아웃 별 필요한 요소 구상하기
  - 최상단 : SafeArea
  - 상단 부분 글자는 가로로 배치 (Row, Text)
  - 그림은 세로로 배치 (Image)
- 요소 별 필요한 위젯 구상하기
  - StorePage(홈화면) ⇒ Column(Row, text), Image
- 디렉토리 분리
  - lib ⇒ main
![](/images/22_기초 앱 구성하기 1/img_01.png)

### 새롭게 사용된 용어

- SafeArea위젯 : 기기 화면의 notch, 상태바 등 을 피해서 UI가 잘리거나 겹치지 않게 배치하는 위젯\
  - 상단 영역은 AppBar를 사용하면 해당 위험 영역은 잡아주기 때문에 SafeArea를 안써도 됨
- Spacer위젯 : Row나 Column위젯 사이의 간격을 조정하는데 사용, 사용 가능한 모든 공간을 차지함
  - Expanded는 자식 위젯이 남은 공간을 차지하지만, Spacer는 남은 공간 빈 공간으로 차지
## Recipe app 화면 구성하기

### 컴포넌트화

- 레이아웃
  - 전체적으로 세로로 배치(Column 사용)
- 나누어진 레이아웃 별 필요한 요소 구상하기
  - 상단 부분 (아이콘 두 개) ⇒ AppBar
  - 두 번째 상단 (”Recipes”) ⇒ RecipeTitle
  - 세 번째 상단 (아이콘 및 글자) ⇒ RecipeMenu
  - 몸통 부분 (그림 및 글자) ⇒ RecipeListItem
- 요소 별 필요한 위젯 구상하기
  - AppBar ⇒ Row, Icons
  - RecipeTitle ⇒ Text
  - RecipeMenu ⇒ Row(Column, Icons, text)
  - RecipeListItem ⇒ Column, Image, text
- 디렉토리 분리
  - lib ⇒ main, AppBar, RecipeTitle, RecipeMenu, RecipeListItem

### 새롭게 사용된 용어

- 폰트 파일 추가
  - Font파일(ttf, otf)을 프로젝트 내 fonts 폴더를 만들어서 추가하기
**pubspec.yaml에 추가한 폰트 선언 **

```dart
# 각각의 font weight에 해당하는 폰트파일을 지정해두면
# Text 위젯의 style에서 fontWeight를 지정하면 해당 fontWeight에
# 해당하는 폰트를 지정해줌
  fonts:
    - family: NotoSansKR
      fonts:
        - asset: fonts/NotoSansKR-Thin.ttf
          weight: 100
        - asset: fonts/NotoSansKR-ExtraLight.ttf
          weight: 200
        - asset: fonts/NotoSansKR-Light.ttf
          weight: 300
        - asset: fonts/NotoSansKR-Regular.ttf
          weight: 400 # FontWeight.normal
        - asset: fonts/NotoSansKR-Medium.ttf
          weight: 500
        - asset: fonts/NotoSansKR-SemiBold.ttf
          weight: 600
        - asset: fonts/NotoSansKR-Bold.ttf
          weight: 700 # FontWeight.bold
        - asset: fonts/NotoSansKR-ExtraBold.ttf
          weight: 800
        - asset: fonts/NotoSansKR-Black.ttf
          weight: 900
```

  - MaterialApp의 theme 속성에 추가 
```dart
MaterialApp(
  theme: ThemeData(fontFamily: 'NotoSansKR'),
  home: const MyHomePage(),
);
```
