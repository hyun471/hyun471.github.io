# 📚 조건문, 반복문

## 조건문

- 프로그램의 흐름을 제어하기 위해 사용한다.
### if문

- if 구성 : if (조건) {코드}
```dart
int a = -1;
if (a < 0) {
	print('a 는 음수입니다.'); // a 는 음수입니다.  (참)
}
```

  - 조건은 참 또는 거짓으로 판별 되어야 하고 조건이 참이면 코드가 실행되고 거짓이면 실행되지 않음
- else if 구성 : if문 + else if (조건) {코드}
```dart
int a = 0;
if (a < 0) {
	print('a 는 음수입니다.');         // 거짓
} else if (a >= 0) {
	print('a 는 0 이거나 양수입니다.'); // a 는 0 이거나 양수입니다.  (참)
}
```

  - if 조건이 거짓이면 else if 문의 조건을 실행 및 판별함
- else 구성 : if + else if + else {코드}
```dart
int a = 3;
if (a < 0) {
	print('a 는 음수입니다.');     // 거짓
} else if (a == 0) {
	print('a 는 0 입니다.');       // 거짓
} else {
	print('a 는 양수입니다.'); // a 는 양수입니다.   (참)
}
```

  - if문 조건과 else if문의 조건이 모두 거짓이면 else 문의 코드가 실행됨
### switch문

- switch 필수 구성 : switch (값) { case 값:  코드 }
```dart
int dayOfWeek = 1;

switch (dayOfWeek) {
  case 1:
    print('오늘은 월요일입니다.');
    break;
  case 2:
    print('오늘은 화요일입니다.');
    break;
  case 3:
    print('오늘은 수요일입니다.');
    break;
  case 4:
    print('오늘은 목요일입니다.');
    break;
  case 5:
    print('오늘은 금요일입니다.');
    break;
  case 6:
    print('오늘은 토요일입니다.');
    break;
  case 7:
    print('오늘은 일요일입니다.');
    break;
  default:
    print('유효하지 않은 숫자입니다.');
}
// 출력 : 오늘은 월요일입니다.
```

  - 특정 값에 대한 경우를 모두 case 문 형태로 나열함
  - 특정 case의 값이 같다면 그 case의 코드가 실행됨
### 조건 표현식

- 조건문 ? 표현식1 : 표현식2
  - 조건문이 참이면 표현식 1을, 조건문이 거짓이면 표현식 2를 따름
```dart
bool isPublic = true;
String visibility1 = isPublic ? 'public' : 'private';   // 조건문 = 참
print(visibility1); // 출력 : public

bool isPrivate = false;
String visibility2 = isPrivate ? 'public' : 'private';  // 조건문 = 거짓
print(visibility2); // 출력 : private
```

- 표현식1 ?? 표현식2
  - 표현식1이 null이 아니면 표현식1, null을 가지면 표현식2를 따름
```dart
int? a = null;
print(a ?? 2);  // a = null
// 출력 : 2

int b = 3;
print(b ?? 2);  // b ≠ null
// 출력 : 3
```

## 반복문

- 어떤 코드를 여러번 반복해서 실행하기 위해 사용한다.
### for문

- 반복할 특정 작업의 반복 횟수가 명확할 때 사용한다.
- for문 필수 구성 : for (초기화식; 조건식; 증감식) {}
  - 조건식이 참이면 증감식이 실행되고 조건식이 거짓이 될때까지 반복
```dart
String message = 'Dart 공부';
for (var i = 0; i < 5; i++) {
	message += '!';
}
print(message); // Dart 공부!!!!!
```

  - 초기화식 변수는 코드블록 안에서 사용이 가능하지만 상수로 선언할 수 없다.
### while문

- while 필수 구성 : while (조건식) {}
  - 조건식이 참이면 while문의 코드블록을 실행하고 거짓이 나올 때까지 반복
```dart
int count = 1;

while (count <= 5) {
	print('count : $count');
	count += 1;
}
/* 출력
count : 1
count : 2
count : 3
count : 4
count : 5
*/
```

### do-while문

- do-while 필수 구성 : do {} while (조건)
  - 일단 do문의 코드를 실행하고 나서 while문의 조건이 참이면 반복 거짓이면 do문을 빠져나옴
```dart
int count = 1;

do {
  print('count : $count');
  count += 1;
} while (count <= 5);
/*
count : 1
count : 2
count : 3
count : 4
count : 5
*/
```

```dart
int count = 6;

do {
  print('count : $count'); // count : 6
  count += 1;
} while (count <= 5);
// count가 6이기 때문에 do문만 실행하고 do문을 빠져나옴
```

### for-in문

- for문의 초기화식의 변수를 사용하지 않고 Collection(List, Set, Map)의 요소를 순회할 때 사용한다.
- for-in 구성 : for (순회식) {코드}
  - in오른쪽의 콜렉션 데이터의 요소가 for문을 돌면서 하나씩 in왼쪽의 변수를 하나씩 대입
```dart
List<String> fruits = ['사과', '바나나', '귤'];

for (var fruit in fruits) {
  print(fruit);
}
/*
사과
바나나
귤
*/
```

### 반복문에 자주 등장하는 키워드

- break : 반복문이 실행되고 있던 중에 반복을 종료할 때 사용한다.
```dart
void main() {
  int count = 1;
  while (count <= 6) {
    count += 1;
    if (count == 5) {
      print('count = $count');
      break;
    }
  }
  print('break 종료');
}
/* 출력 : count = 5
          break 종료
*/
```

- continue : 반복문 실행 중 특정 조건을 건너뛰고 싶을 때 사용한다.
```dart
void main() {
  final fruits = ['사과', '파인애플', '오렌지', '귤', '복숭아'];
  for (final fruit in fruits) {
    if (fruit == '파인애플') {
      continue;
    }
    print('맛있는 $fruit');
  }
}
/*
맛있는 사과
맛있는 오렌지
맛있는 귤
맛있는 복숭아
*/
```

- return : 반복문 실행 중 즉시 함수를 종료하고 값을 반환할 때 사용한다.
  - break와 차이는 break는 반복을 종료하고 return는 함수를 종료한다.
```dart
void main() {
  int count = 1;
  while (count <= 6) {
    count += 1;
    if (count == 5) {
      print('count = $count');
      return;
    }
  }
  print('return 종료');
}

// 출력 : count = 5
```

## 주석

- 코드 안에 코드가 무슨 의미인지 코드에 대한 설명이다.
```dart
int a = 3; // 여기서 int a = 3; 까지만 읽음
/*
여기에 해당하는
두줄 모두 컴파일러가 읽지 않음
*/
/// 이 함수는 두 수를 더합니다.
int add(int a, int b) {
  return a + b;
}
```
