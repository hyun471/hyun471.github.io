# 💡 Console Shopping mall 심화 도전

## Console Shoppingmall 프로젝트 심화 도전 기능

- 쇼핑몰 프로그램을 종료할 시 한번 더 종료할 것인지 물어보는 기능
  - 4번을 입력하면 정말 종료하시겠습니까? [ y / n ] 출력 이후 y는 종료, n는 다시 shoppingmall meau 출력
- 장바구니를 초기화할 수 있는 기능
  - 5번을 입력하면 장바구니가 초기화됩니다. 출력과 함께 장바구니가 초기화됩니다.
- 장바구니에 담은 상품들의 목록과 가격을 볼 수 있는 기능
  - 3번을 입력하면 장바구니에 담긴 상품의 목록과 가격의 총 합을 계산하여 출력
  - 만약 장바구니가 비어있다면 담긴 상품이 없다고 출력
### 심화 기능 도전 전 수정 사항

- 기존 필수 요구에서 요구한 장바구니 추가 함수와 장바구니에 있는 상품 가격의 합 함수를 클래스 Shoppingmall에 추가하여 코드를 깔끔하게 정리
  - 옮기는 과정에서 데이터를 주고받는 변수들을 다시 정비
- 전체 기능을 확인해 보니 가독성을 위하여 4번에 장바구니 초기화를 5번에 프로그램 종료로 순서를 바꾸기 위하여 While 반복문에 case 수정
### 1. 프로그램 종료 시 한번 더 여부 확인 기능 구현

- 먼저 기존에 작성한 코드에서 조건문을 달아 입력 값에 따른 종료 여부를 추가해줘야겠다.
```dart
case "5":
  print("이용해 주셔서 갑사합니다. 안녕히 가세요~");
  runningLoop = false;
  break;
```

- 먼저 case 5번에 들어오게 되면 “정말로 종료하시겠습니까? [ Y / N ]”라는 문구가 출력되어야 하고 입력 값을 입력할 수 있게 만들어야 한다.
```dart
case "5":
        // 쇼핑몰 프로그램 종료 시 한번더 확인 기능
        print("정말로 종료하시겠습니까? [ Y / N ]");
        String? exitRun = stdin.readLineSync();
```

- 이후 if문을 사용하여 입력 값의 값이 Y면 잘가 라는 문구와 함께 프로그램 종료, 입력값이 N이면 다시 쇼핑몰 입장하게 만들어야 한다.
```dart
if (exitRun == "Y") {
   print("이용해 주셔서 갑사합니다. 안녕히 가세요~");
   runningLoop = false;
   break;
 } else if (exitRun == "N") {
   print("콘솔 쇼핑몰에 오신걸 환영합니다.");
   break;
```

### 2. 장바구니 초기화 기능

- 장바구니를 초기화 하기 위하여 초기화 기능을 실행해줄 함수를 만들어야겠다.
```dart
void clearCart() {
  
}
```

- 이제 초기화를 위하여 장바구니 안에 담긴 내용물을 지워주거나 없던 리스트로 덮어주면 될 거 같다.
- 내용물을 지우기엔 너무 복잡한거 같아 빈 리스트로 덮어 한번에 지우기로 하였다.
  - cart = {};를 활용하면 빈 리스트를 덮어주는 줄 알았는데 이 기능은 cart에 빈 리스트를 추가하는 것이다. 즉, 장바구니에 담긴 리스트는 사라지는게 아니라 데이터로만 남아있게 된다.
  - 하지만, dart와 java와 같은 고급 언어는 이런 안 쓰는 데이터들을 자동으로 지워 메모리가 과부화  되지 않게 해준다고 한다.
  - cart = {};사용이 가능하고 c언어 같은 경우 cart.clear로 데이터를 지우지만 이것 또한 많은 cpu를 잡아먹어 여기 dart 언어에서는 cart = {}; 써도 크게 차이가 없다.
```dart
void clearCart() {
  cart = {};
  print("장바구니를 초기화 합니다.");
}
```

- 다음 장바구니가 비어있으면 이미 장바구니가 비어있다고 표시를 해줘야한다.
- if문을 활용하여 장바구니의 리스트 데이터의 존재 여부로 비어있는지 아닌지 판단하기로 하였다. 
```dart
void clearCart() {
    // 조건문을 사용하여 장바구니 담긴 상품이 있으면 초기화
    if (cart.isNotEmpty) {
      cart = {};
      print("장바구니를 초기화 합니다.");
    } else if (cart.isEmpty) {
      print("이미 장바구니가 비어있습니다.");
    }
  }
```

### 3. 장바구니에 담은 상품들의 목록과 가격의 합계를 출력하는 기능

- 기존에 3번을 만들 때 이 도전 기능 부분을 생각하고 만들었다. 
- 이전 총 가격을 불러올 때 cart에 저장되어 있는 Map 데이터를 불러와 가격 부분과 수량부분을 곱하여 새로운 변수에 저장하고 이를 출력하였다. 
- 이번에도 새로운 리스트 변수를 만들어서 cart에 저장되어 있는 이름 데이터를 불러와 저장해주면 되겠다고 생각했다.
```dart
void showTotal() {
    int totalPrice = 0;
    List<String> addProduct = [];
    if (cart.isEmpty) {
      print("장바구니에 담긴 상품이 없습니다.");
      return;
    } else {
      for (var entry in cart.entries) {
        Product product = entry.key;
        int count = entry.value;
        totalPrice += product.price * count;
        // 장바구니에 담긴 물품 출력
        addProduct.add(product.name);
      }
      print("장바구니에 담긴 상품 ${addProduct.join(', ')} 이며, 총 가격은 $totalPrice원 입니다.");
      return;
    }
  } // 장바구니에 담긴 총 가격
```

## 트러블슈팅 과정

### 장바구니에 담은 상품의 상품명을 새로운 리스트에 저장 과정

- 새로운 리스트 변수에 상품 이름을 담는 것이 생각보다 어려웠다. 
- 총 가격을 나타내는 변수처럼 연산자를 활용하여 나타내면 되겠다고 생각했지만 연산이 제대로 되지 않았다. 
- 새로운 리스트 addProduct를 이미 위에서 선언햇는데 for문 안에서 선언한다고 판단되어 물품의 이름이 제대로 들어가지 않는 현상이 발생한 것이다.
- 이를 해결하기 위하여 튜터님의 도움을 받아보니 이미 선언된 변수에 add함수를 메서드 체이닝 하여 매개변수를 product.name으로 하면 매개변수 product.name이 add 함수를 통해 addProduct 변수에 저장되는 것을 알게 되었다.
- 선언된 변수를 함수와 엮는 생각이 필요하다고 생각된다. (이것은 많은 연습이 필요할 거 같다.)
### 장바구니 담기에서 새로운 리스트에 저장된 상품명을 출력 값으로 불러오는 과정

- 새로운 리스트 addProduct에 장바구니에 담긴 상품명을 String타입으로 저장하는데 성공했다.
- 이제 이 addProduct를 print하면 되는 줄 알았다.
- 하지만, print(”상품은 $addProduct 입니다.”) 하게 되면 “상품은 [Shirt, Socks] 입니다.” 로 리스트 형태로 나오게 되었다. 이대로되 상관 없지만 좀더 편하게 보기 위하여 리스트를 제거하기로 했다.
- join는 List의 요소들을 하나의 문자열로 합쳐주는 기능으로 addProduct.join(’, ‘)으로 상품명을 문자열 형태로 나열하게 만들었다.
### 장바구니 초기화 과정

- 장바구니 초기화 과정 중 if문의 조건을 (cart == {})로 구성하면 cart가 빈{}라면 이라는 뜻이 되는 줄 알았다.
- 하지만 cart에 빈 리스트 데이터를 계속 생성하여 이미 장바구니를 비웠어도 계속 초기화를 하는 현상이 발생하였다. 
- 데이터가 비어있는지 아닌지를 확인하기 위해서는 isNotEnpty, isEmpty 라는 함수를 활용하여 메서드 체이닝으로 만들 수 있었다.
### 프로그램 종료 여부 재확인

- 프로그램 종료 여부를 재확인에서 “정말로 종료하시겠습니까? [ Y / N ]”의 출력값 다음으로 입력값을 Y나 N을 하면 종료 혹은 쇼핑몰 사용 계속으로 넘어가는데 입력값을 소문자로 입력하게 될 경우 인식을 못한다.
- 이를 해결하기 위하여 튜터님께 도움을 청해보니 toUpperCase()라는 함수를 알려 주셨다.
- toUpperCase()라는 함수는 문자열에 있는 모든 소문자를 대문자로 변환해 주는 함수이다.
- 변수의 타입이 String? 이므로 null일수 도 있다는 표시인 변수에 ?를 붙인 다음 메서드 체이닝을 통해  exitRun?.toUpperCase() 으로 구현해야한다.
- toUpperCase()함수의 반대 의미인 toLowerCase()라는 함수도 존재한다.
