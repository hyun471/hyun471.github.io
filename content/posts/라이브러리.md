# 라이브러리

# 📚 라이브러리(Library)

## 라이브러리

- 변수, 함수, 클래스 등을 모아서 하나의 파일 또는 파일 집합으로 만든 코드 집합이다.
- 코드를 분리하고 관리하기 쉽게 하기 위해 사용된다.
- 필요한 코드만 뽑아 사용해서 빠르게 구현하고 재사용성이 높으며 코드 가독성이 좋다.
### 라이브러리 종류

**Dart SDK에 포함된 라이브러리 (1st-party)**

- 프로그래밍에 필요한 핵심 라이브러리를 Dart 팀이 직접 만들고 Dart 설치 시 기본으로 제공됨
**플랫폼에 따라 사용할 수 있는 라이브러리가 다름**

**모든 플랫폼 (Native + Web)**

- dart:core
- dart:async
- dart:collection
- dart:convert
- dart:developer
- dart:math
**Native platform (Native는 Mobile, Desktop이 주 타켓)**

- dart:ffi
- dart:io
**Web platform (Web은 JavaScript로 컴파일되는 코드임)**

- package:web
- dart:js_interop
- dart:html
**Dart SDK에 포함되지 않은 라이브러리 (3rd-party)**

- Dart에서 제공하는 표준 라이브러리가 아닌 외부 개발자들이 만든 라이브러리
- Pub.dev를 통해 다양한 라이브러리를 찾고 적용할 수 있음
### 라이브러리 사용법

- import라는 키워드를 통해 라이브러리를 불러온다.
  - import '[라이브러리 이름]'; : 외부 pub.dev에서 라이브러리를 불러오는 경우
  - import '[파일 경로]'; : 내부 파일에서 라이브러리를 불러오는 경우
- 불러온 라이브러리의 안에 코드는 자유롭게 사용이 가능하다.
**import에서 자주 사용되는 옵션**

- as 를 통해 라이브러리에 별칭을 부여할 수 있음
- show 를 통해 라이브러리에서 필요한 부분만 선택적으로 가져올 수 있음
- hide 를 통해 라이브러리의 특정 부분을 제외하고 가져올 수 있음
- deferred as 를 통해 라이브러리가 필요한 시점에 로드되도록 할 수 있음
