## 동기/비동기

- 동기 방식은 작업이 순차적으로 처리되는 것을 의미한다.
- 비동기 방식은 작업이 병렬적으로 처리되는 것을 의미한다.
![](/images/02_클래스/img_01.png)

- 동기 예시
```dart
void main() {
  print('작업 1 시작');
  performTask();
  print('작업 1 완료');
}

void performTask() {
  print('작업 2 실행');
}

출력
작업 1 시작
작업 2 실행
작업 1 완료
```

- 비동기 예시
```dart
void main() {
  print('작업 1 시작');
  performTask();
  print('작업 1 완료');
}

Future<void> performTask() async {
  await Future.delayed(Duration(seconds: 2));
  print('작업 2 실행');
}

출력
작업 1 시작
작업 1 완료
(2초뒤에)
작업 2 실행
```

### 유의할점

- 동기 예시에서 순차적으로 진행하면 ‘작업 1 시작’ → ‘작업 1 완료’ → ‘작업 2 실행’ 
순으로 실행될 것 
같지만 밑에 선언된 performTast 함수의 의해 ‘작업 1 시작’  다음 performTask 함수가 
실행되어 예시 출력과 같이 나오는 것을 유의해야 한다.
- 동기와 비동기는 같아 보이지만 비동기 예시는 결국 2초라는 지연 시간이 병렬적 
구조를 만들게 된다.
## 클래스

- 클래스(class)는 어떤 특성과 기능을 가진 객체를 사용할 수 있게 만드는 설계도와 같다. 
- 클래스는 class + [클래스명] + { 생성자() + 맴버변수 + 메서드() {} }의 형태로 작성된다.
  - 객체 : 클래스를 통해 실제로 작동하는 것
  - 생성자 : 클래스와 같은 이름을 사용하며 객체를 만들 때 자동으로 실행됨
  - 맴버변수 : 변수와 쓰임이 같으며 클래스 안에서 선언되면 맴버변수라고 칭함
  - 메서드 : 함수와 기능이 같으며 클래스 안에서 메서드라고 칭함
- 클래스 예시
```dart
//클래스 = Dog
class Dog {
  // 멤버변수 (특성)
  String name = '초코';
  int age = 2;

  // 생성자
  Dog(this.name, this.age);

  // 메서드 (기능)
  void bark() {
    print('$name: 멍멍!');
  }
}

void main() {
  var dog = Dog('코코', 3);
  dog.bark(); 
}

//출력 코코: 멍멍!
```

### 유의할점

- 클래스를 선언한다고 해서 메서드가 자동으로 실행이 되지 않는다. 클래스는 설계도와 
같으므로 실행하기 위해선 명시적으로 객체에서 직접 호출을 해야 실행된다.
- 클래스는 타입으로 사용이 가능하다.
```dart
Dog myDog; // Dog는 타입
myDog = Dog('초코', 2); // Dog 클래스의 객체(인스턴스)를 생성해서 대입
```

  - 예시
```dart
//클래스 = Dog
class Dog {
  // 멤버변수 (특성)
  String name = '초코';
  int age = 2;

  // 생성자
  Dog(this.name, this.age);

  // 메서드 (기능)
  void bark() {
    print('$name: 멍멍!');
  }
}

Dog dog1 = Dog('코코', 3);
Dog dog2 = Dog('초코', 2);

void main() {
  dog2.bark(); 
}

//출력 초코: 멍멍!
```

## 위젯트리

- 위젯 트리는 앱 화면에서 버튼, 텍스트, 아이콘 등의 위젯이 겹겹이 쌓여 만들어진 구조이다.
- 트리처람 가지를 뻗어나가는 구조로 한 부모 위젯이 여러 자식 위젯을 가질 수 있다.
![](/images/02_클래스/img_02.png)

### 앱 구조(위젯트리) vs 프로젝트 구조(파일) 

- 앱 구조
  - Header, Home과 같은 위젯은 실제로 화면에 보여지는 UI의 한 부분
  - 파일 안에 작성된 클래스
- 프로젝트 구조
  - home.dart, main.dart 같은 파일들은 앱을 만드는 작업공간
  - 코드가 여러개 있으며 관리하기 쉽도록 나누는 것으로 ‘설계도 보관함’ 같은 개념

## Dart 문법 기초 정리

- 지금까지 배운 타입, 변수, 함수, 클래스, 생성자, 함수, 위젯, 위젯트리, 파일을 비유를 통해 정리해 보자
  - 표로 정리
  - 코드로 정리
```dart
[집 (위젯 트리)]
 ├── 거실 (위젯 인스턴스)
 │     ├── 창문 (위젯 인스턴스)
 │     │     ├── 유리 (변수: 재질)
 │     │     ├── 나무틀 (변수: 재질)
 │     │     ├── 여닫이 기능 (함수: open(), close())
 │     │     └── 생성자 (창문을 만드는 방법)
 │     └── 소파 (다른 위젯 인스턴스)
 ├── 안방 (위젯 인스턴스)
 └── 화장실 (위젯 인스턴스)
```
