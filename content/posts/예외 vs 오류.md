## 예외

- 프로그램 실행 중에 발생하는 에러 상황이다.
  - 없는 파일 열기, null 값을 잘못 참조하기 등
- 예외를 처리해주지 않으면 프로그램이 즉시 종료된다.
- 모든 예외는 Exception 클래스를 상속 받아서 구성된다.
### 예외에서 자주 사용하는 키워드

**발생되는 예외를 처리할 때 사용하는 키워드**

**try : {}블록 안의 코드가 실행되다가 오류가 발생하면 바로 예외 처리 절차로 넘어감**

```dart
try {
  // 예외 발생 가능성 있는 코드
}
```

**catch : 어떤 예외든 예외가 발생했을 때 그 예외 객체를 받아 처리할 때 사용**

```dart
catch (e) {                // 여기서 e는 실제로 발생한 예외 객체
  print("예외 발생: $e");
}
```

**on : 예외 타입을 명시해서 특정 예외 타입만 잡아서 처리할 때 사용 (예외 타입 = 예외 클래스)**

```dart
on FormatException {
  print("형식 오류 처리");
}
```

**finally : 예외 발생 여부와 관계없이 무조건 실행되는 블록 (예 : 리소스 해제, 로그 남기기 등)**

```dart
finally {
  print("항상 실행됨");
}
```

**예외를 직접 발생하는 키워드 : throw**

- 구성 : throw + 예외 클래스 
**예시 코드**

```dart
void checkAge(int age) {
  if (age < 0) {
    throw Exception("나이는 음수가 될 수 없습니다");  // 직접 예외 발생
  }
  print("나이: $age");
}

void main() {
  try {                       // 예외가 발견되면 예외 처리로 넘김
    checkAge(-3);
  } catch (e) {               // catch로 예외 잡고 처리하기
    print("에러 발생: $e");
  }
}
```

## 오류

- 프로그램 실행 중 발생하는 심각하고 복구 불가능한 문제로 그대로 프로그램을 종료 시킨다.
- 모든 오류들은 Error 클래스에서 상속 받아 만들어진 객체이다.
- 오류가 발생하게 되면 런타임 도중에 자동으로 오류 클래스 객체를 만들어 오류 내용을 출력한다.
### 오류 종류

- 오류는 발생을 예상할 수도 없기 때문에 처리할 수 없어 오류가 출력되는 그때 그때 처리한다.
- Dart에서 정의된 오류를 커스텀 하고 싶을 때 Error클래스에서 상속 받아 재정의하여 사용한다.
