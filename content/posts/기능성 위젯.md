# 📚 위젯의 종류 3

## 기능성 위젯

### Text 위젯

- Text 위젯은 텍스트(문자열)를 화면에 표시하는 위젯이다.
  - Text 위젯은 Text(’문자열’)로 구성된다.
- 자주 사용되는 style 옵션
  - style: TextStyle( 옵션 )로 구성하여 사용한다
```dart
Text(
  '안녕하세요',
  style: TextStyle(
    fontSize: 20,                            // 글자 크기를 20으로 설정
    fontWeight: FontWeight.bold,             // 글자 굵은 글씨로 설정
    color: Colors.blue,                      // 글자 색을 파란색으로 설정
    letterSpacing: 2.0,                      // 글자 사이 간격을 2.0으로 설정
    wordSpacing: 4.0,                        // 단어 사이 간격을 4.0으로 설정
    decoration: TextDecoration.underline,    // 글자에 밑줄
    fontFamily: 'Roboto',                    // Roboto 폰트 사용
  ),
)
```

### TextField 위젯

- TextField위젯은 Flutter에서 사용자가 글자를 입력할 수 있는 입력창을 만들어주는 위젯이다.
  - 구성은 TextFielid()로 구성된다.
- TextField위젯에서 자주 사용되는 옵션
```dart
final myController = TextEditingController();

TextField(
  maxLength: 10,                    // 최대 10글자까지만 입력 가능
  readOnly: true,                   // 사용자 입력을 막음
  textAlign: TextAlign.center,      // 가운데 정렬
  decoration: InputDecoration(      // 입력창의 레이블, 아이콘, 테두리 등 꾸밈
    labelText: '아이디',
    hintText: '아이디를 입력하세요',
  ),
  controller: myController,         // 입력창 글자를 직접 제어하는 도구
)
```

  - InputDecoration()은 매우 자주 사용되는 입력창을 꾸미는 클래스
  - 자주 쓰이는 속성들
```dart
labelText: '이메일'                          // 입력창 위쪽에 보이는 설명표
hintText: 'example@gmail.com'                // 희미하게 보여주는 힌트
helperText: '이메일을 입력하세요'             // 입력창 아래에 도움말 텍스트
errorText: '이메일 형식이 올바르지 않아요'     // 유효성 검사 시 오류 메세지
border: OutlineInputBorder()                 // 테두리 모양 설정 
```

### Switch 위젯

- Switch위젯은 사용자가 on/off 상태를 전환할 수 있게 해주는 토글 스위치를 생성하는 위젯이다.
- Switch위젯의 필수 구성요소는 Switch() + value + onChanged(반필수)로 구성된다.
```dart
Switch(
  value: true,                       // 스위치 온/오프 (true/false로 작동)
  onChanged: (bool newValue) {       // onChanged가 없을시 읽기 전용
    // 상태가 바뀔 때 실행할 코드
  },
)
```

### Slider 위젯

- Slider위젯은 사용자가 지정된 범위 내 값을 선택할 수 있는 슬라이더를 만드는 위젯이다.
- 주로 밝기 조절, 음량 조절에 사용된다.
- Silder위젯의 필수 구성요소는 Silder() + value + onChanged(반필수)로 구성된다.
```dart
Silder(
  value: 0.5,                          // 현재 슬라이더 위치 (보통 0.0 ~ 1.0 사이값)
  onChanged: (double newValue) {       // onChanged가 없을시 읽기 전용
    // 슬라이더 값이 바뀔 때 실행할 코드
  },
)
```

- Slider에서 자주 사용되는 옵션
```dart
Slider(
  value: 0.5,           // 필수, 현재 위치
  onChanged: (newVal) { // 필수는 아니지만 거의 사용, 값이 바뀔 때 실행할 함수
    print(newVal);
  },
  min: 0.0,             // 옵션, 최소값
  max: 10.0,            // 옵션, 최대값
  divisions: 5,         // 옵션, 단계 수 (0, 2, 4, 6, 8, 10 이런 식으로 나누기)
  label: '볼륨',        // 옵션, 값 위에 표시되는 텍스트
)
```

### Image 위젯

- Image위젯은 다양한 소스에서 이미지를 불러와 화면에 표시하는 위젯이다.
- Image위젯의 기본 구조는 Image.asset('assets/images/picture.png')로 구성된다.
  - 즉, asset에 포함되어 있는 이미지를 경로로 불러온다.
  - asset에 이미지가 없으면 이미지를 asset 폴더에 넣고 yaml에 이미지 경로를 등록해 줘야한다.
```yaml
// pubspec.yaml에 이미지 경로 등록하기

flutter:
  assets:                                // pubspec.yaml에서 assets부분에 넣기
    - assets/images/picture.png          // 이부분만 새롭게 넣어야 함
    - assets/images/                     // 폴더를 넣으면 안에 이미지가 모두 포함
    
    
//꼭 Tab을 눌러 들여쓰기를 해야 작동함(이거로 오류 많이 발생함)
```

  - 인터넷에 있는 이미지를 불러올 때 Image.network('https://example.com/image.png') 경로를 사용한다.
  - 기기에 저장된 이미지 파일을 불러올 땐 Image.file(file)을 사용한다.
- Image위젯에 자주 사용되는 옵션
```dart
Image.asset(
  'assets/images/picture.png',
  width: 200,                                // 이미지의 가로 크기 지정
  height: 100,                               // 이미지의 세로 크기 지정
  fit: BoxFit.cover,                         // 이미지를 박스 안에 어떻게 맞출지 지정
  alignment: Alignment.center,               // 이미지 위치 지정
)
```

# 💡 문제 해결

## 학습하면서 생각이 든 의문점 및 해답(ChatGPT)

- GestureDetector 클래스에서 behavior의 종류중 opaque에 대한 궁금증
**Q1. 투명한 박스 위젯의 경우 behavior가 없어도 작동되는 것 아닌가?**

  A1. 투명한 박스의 경우 감지를 할 수 가 없다. 
      만약 박스가 빨간색이면 behavior를 생략해도 기본값인 deferToChild가 자동으로 
      들어가며 gesture의 대상이 100x100 빨간 박스라고 인식이 가능하다. 반면, 투명한 박스는 
      gesture가 100x100의 투명 박스를 인식할 수 없어 behavior가 필요하다.

```dart
//빨간 박스 경우
GestureDetector(
  onTap: () {
    print("탭!");
  },
  // behavior: 생략 (기본값 deferToChild)
  child: Container(
    width: 100,
    height: 100,
    color: Colors.red,
  ),
)
```

```dart
//투명한 박스의 경우
GestureDetector(
  onTap: () {
    print("탭!");
  },
  behavior: HitTestBehavior.opaque,
  child: Container(
    width: 100,
    height: 100,
    color: Colors.transparent,
  ),
)
```

  - 잘 작동하는 걸 확인하였다. 
  - 그럼 ChatGPT가 추천해 주는 코드를 확인해보자 
```dart
import 'package:flutter/material.dart';

void main() {
  runApp(const MyApp());
}

class MyApp extends StatefulWidget {
  const MyApp({super.key});

  @override
  State<MyApp> createState() => _MyAppState();
}

class _MyAppState extends State<MyApp> {
  final TextEditingController myController = TextEditingController();

  @override
  void dispose() {
    myController.dispose(); // 꼭 메모리 정리해줘!
    super.dispose();
  }

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        body: Center(
          child: TextField(
            controller: myController,
            decoration: InputDecoration(
              labelText: '비밀번호',
              hintText: '비밀번호를 입력하세요',
              prefixIcon: Icon(Icons.lock),
              border: OutlineInputBorder(),
              suffixIcon: IconButton(
                icon: Icon(Icons.clear),
                onPressed: () {
                  myController.clear(); //
                },
              ),
            ),
          ),
        ),
      ),
    );
  }
}
```

  - 1번 메모리 정리를 위해 dispose를 추가함, 2번 const를 추가하였다.
**1번 메모리 정리를 위해 dispose를 추가한 이유**

    dispose()는 사용이 끝난 메모리를 정리하는 함수이다. 앱에서 사용이 끝나도 메모리를 정리하지 않으면 계속 남아있게 되어 앱이 점점 느려지고 메모리 누수가 생길 수 있다. 

**2번 const 추가한 이유**

    const는 컴파일 타임에 미리 인스턴스를 생성하여 성능을 더 최적화 할 수 있다. 
특히 Flutter는 UI를 자주 새로 그려, 변하지 않는 위젯에 const를 붙이면 낭비를 줄일 수 있다.

    여기서 입력창의 입력값을 지우는 것 외에는 변하지 않으니까 const를 붙이는 것을 추천한다.

**Q1. 그럼 대부분 앱에선 동적으로 작동을 많이 하니까 const MyApp()에서 const를 
       빼야하는 것 아닌가?**

    A1. 대부분의 경우, MyApp()앞에 const를 붙이는 게 더 좋고, 일반적으로도 붙인다.
      대부분 앱은 시작 시 테마, 초기 구조 등 고정된 설정값을 가지기 때문에 const
      MyApp()으로 만들어 효율적으로 시작하는 것을 추천한다.
      즉, MyApp에 const를 붙이는 것은 앱의 “시작 부분”은 대부분 변하지 않기 때문이다.

    - const를 못 붙이는 경우
      - MyApp의 생성자에 const가 없는 변수를 넣을 때
      - 즉, MyApp이 런타임에 따라 다르게 구성될 때(예: 유저 상태, 다크모드 설정, 외부 값 등)
