# 📚 Dart 문법 기초 1

## 변수

- 변수에는 int, String, bool 등 Dart의 기본 타입이 있다.
- 타입 + 변수 = 값 의 형태로 나타낸다.
- 타입은 Dart 문법에서 정해져 있다.
- 변수는 임의로 이름을 정하지만 보통 의미하는 바를 알아보기 쉽게 정한다.
- int? 에서 ?는 null을 나타내며 즉, int? 는 null을 허용하는 정수형 타입이다.
- 여러가지 타입 종류 및 기본 구조
```dart
int count = 10;                  // 정수
double temperature = 36.5;       // 실수(소수점 숫자)
num number = 3.14                // 모든 수
String title = "Hello Dart";     // 문자열
bool isDarkMode = false;         // 참/거짓
List<String> fruits = ["사과", "바나나", "포도"]; // 리스트
```

### 숫자 타입 int, double, num의 관계

- num은 int와 double의 상위 타입이다. 즉, num 타입을 쓰게 되면 모든 숫자가 가능하다.
  - 코드를 수정하거나 보완할 때 정확한 타입을 써서 의도를 분명하게 하는 게 코드를 읽을 때도 이해하기 쉽다.
  - 실제로 코드를 작성할 때 보통 정확한 타입을 쓰는 게 좋다.
## 함수

- 특정 작업을 수행하는 코드 묶음이다. 
- 함수 선언은 타입 + 함수이름 + (매개변수) + 코드블록 의 형태로 작성한다.
- 함수 또한 임의로 이름을 정하지만, Dart에서 제공하는 정해진 함수도 있다.
- 함수 선언의 기본 구조
```dart
타입 함수이름(매개변수들) {
  // 실행할 코드
  return 반환값;  // 반환값이 없으면 return 생략 가능 (void 타입)
}
```

- 예시
```dart
void sayHello() {               //반환하지 않고 sayHello라는 함수를 실행
  print("안녕하세요");           //sayHello는 매개변수가 없고 {}의 코드를 실행
}                               //{} 코드는 "안녕하세요" 문자열을 출력

sayHello();                //sayHello함수를 실행 -> "안녕하세요" 문자열 출력
```

### 함수 이름 뒤의 () 의미

- ()는 함수를 실행하라 라는 뜻이다.
- ()없이 함수만 쓰게 되면 함수 자체를 가리키는 것이 된다.
- 즉, 함수가 아니라 함수가 저장된 ‘참조(주소)’를 나타내는 값이다. 
- ()안에 매개변수가 있으면 매개변수를 가지고 실행하라는 뜻이다.
### 함수의 장점

- 같은 작업을 여러 번 반복할 필요가 없다.
- 이름만 부르면 쉽게 실행이 가능하다.
- 코드를 깔끔하고 재사용이 가능하도록 만들어 준다.
# 💡 문제 해결

## 오늘 해결한 기술적 문제

- 본격적인 짝맞추기 게임을 만들기에 돌입하면서 모르는 문제가 발생했다.
- 코드를 보다 보니 처음 보는 어려운 것이 눈에 들어왔다.’
```dart
@override
Widget build(BuildContext context) {
  return SingleChildScrollView(
    child: Wrap(
      spacing: 4,
      runSpacing: 4,
      children: [
        for (var i = 0; i < 12; i++) CardWidget(),
      ],
    ),
  );
}
```

### @override 이게 뭘까?

- @override는 부모 클래스(상위 클래스)의 메서드를 ‘재정의(override)’할 때 쓰는 표시이다.
- 즉, 함수 (ex : build())를 사용자가 직접 원하는 대로 고쳐서 사용할 수 있게 해주는 표시이다.
- 그래서 위에 @override를 붙이면 ”기본적인 함수를 내가 새롭게 정의할게” 라고 알려주는 것이다.
